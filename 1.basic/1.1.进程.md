
## 进程
### 1 概念
进程（Process）就是“正在运行的程序实例”。

比如你双击打开微信、浏览器、IDE，每打开一个，操作系统通常都会创建一个进程。

核心点可以这样理解：
- 进程是操作系统进行资源分配的基本单位。
- 每个进程有自己的内存空间（代码、数据、堆、栈）、打开的文件、PID 等。
- 进程之间默认彼此隔离，稳定性好（一个进程崩了，不一定拖垮别的进程）。
- 进程间通信要靠 IPC（管道、消息队列、共享内存、socket 等）。
- 进程切换开销通常比线程更大。

和线程一句话对比：
- 进程：重、隔离强。
- 线程：轻、共享同一进程内存。

可以把“进程”整理成一句话：

`进程 = 操作系统分配资源后，正在运行的一个程序实例（有独立 PID、独立内存）`。

### 2 如何判断不同的进程
判断是不是不同进程，最直接看 `PID` 是否不同。

>Example 1：两个命令就是两个进程

```py
# show_pid.py
import os, time
print("PID:", os.getpid())
time.sleep(30)
```

开两个终端分别跑：

```bash
python show_pid.py
python show_pid.py
```

你会看到两个不同的 `PID`，这就是两个不同进程。

>Example 2：一个脚本里再开子进程

```python
# parent_child.py
import os, subprocess, sys

print("parent PID:", os.getpid())
subprocess.run([sys.executable, "-c", "import os; print('child PID:', os.getpid())"])
```

运行一次：

```bash
python parent_child.py
```

输出里 `parent PID` 和 `child PID` 不同，说明父子是不同进程。
